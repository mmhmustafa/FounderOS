# Runtime Planner Engine

> **Status:** Deterministic in-memory implementation complete
>
> **Implementation:** `src/founderos_runtime/execution_context.py`, `planning_rules.py`, and `planner.py`

## Purpose

The Runtime Planner is the first orchestration layer above the Runtime Foundation. It inspects an immutable ExecutionContext and produces an immutable ExecutionPlan describing the recommended workflow, required and missing artifacts, agent roles, allowed transitions, quality gates, and blocking reason.

The Planner recommends work. It does not execute work or mutate runtime state.

## Inputs

ExecutionContext contains:

- Project ID and current state
- Approved completed artifact types
- Pending artifact types
- Active Agent and Workflow definitions
- Project Decisions and risks
- Ordered Event summaries
- Current next action

`ExecutionContextBuilder` constructs this context using defensive repository reads.

## Outputs

ExecutionPlan contains:

- Current state
- Recommended workflow
- Required and missing artifacts
- Recommended agent roles
- State Machine-allowed transitions
- Blocking reason
- Quality-gate requirements
- Next-state candidate
- Deterministic confidence score

## Components

### ArtifactPlanner

Compares state-specific artifact requirements with approved Project artifacts. Draft, rejected, deprecated, missing, and cross-project artifacts do not count as completed evidence.

### WorkflowSelector

Selects the canonical workflow for the current state, verifies the proposed next state is allowed by the State Machine, and returns a clear block when required approved artifacts are missing.

### AgentRouter

Maps the selected workflow to a stable, ordered tuple of required agent roles. It never invokes an agent or model.

### Planner

Combines context, artifact analysis, workflow selection, agent routing, and State Machine route requirements into one ExecutionPlan.

## Invariants

1. Planning is read-only and creates no Project, Event, Run, Transition, or other record.
2. Identical ExecutionContext values produce identical ExecutionPlan values.
3. Unknown states fail explicitly.
4. The Planner cannot recommend a next-state candidate forbidden by the State Machine.
5. Missing required approved artifacts create a blocked plan.
6. Recommendations do not imply approval, execution, or a successful transition.
7. Lifecycle content is never generated by the Planner.

## Dependencies

- RuntimeRepositories for context construction
- StateMachine read-only route inspection
- Approved artifact metadata
- Static deterministic planning rules

## Limitations

- Workflow and agent mappings are static code metadata in this milestone.
- Available definitions are reported in context but are not yet required to exist before a recommendation is made.
- Confidence is `1.0` for recognized deterministic rules; it is not an evidence confidence score.
- The Planner does not choose among multiple valid recovery branches; it reports all allowed transitions and one primary next-state candidate.
- No workflow execution, content generation, AI call, persistence adapter, CLI, or UI exists.

## Tests

`tests/test_planner.py` verifies required early lifecycle routes, blocking behavior, unknown-state rejection, plan completeness, deterministic output, context inventory, and zero repository mutation.
